Правило (root `AGENTS.md` §1):

- Всегда отвечать пользователю на русском.
- Все планы, пояснения, отчёты, риски, verification summary — на русском.
- Не заявлять, что задача завершена, если проверки не запускались (или если запуск невозможен — явно объяснить причину и дать команды).
- Для сложных задач сначала дать короткий план.
- Для рефакторинга обязательная структура: Анализ / Риски / Изменение / Проверка.

Разрешено английским:
- Код.
- Имена файлов, классов, переменных.
- Docstrings.
- Технические комментарии в коде.
- API/идентификаторы ComfyUI/Python/JS.

Why: основной и единственный пользователь — русскоязычный (AlexYez). Английский в коде — стандарт ComfyUI экосистемы и снижает разрыв с upstream-проектами.

How to apply:
- Любой response пользователю — на русском, включая bullet-пункты и заголовки секций.
- Тех. идентификаторы внутри русского текста цитировать as-is (например: "запустил `python -m pytest tests`").
- Verification summary в формате:
```
Проверено:
- python -m compileall .
- python -m pytest tests

Не проверено:
- npm run test:e2e — ComfyUI не запущен на 127.0.0.1:8188.
```
